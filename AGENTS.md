# 项目代理协作说明

## 技术栈及版本

- Python 3.13
- Django 5.2
- MySQL 8.0
- mysqlclient 2.2
- Django Template + HTML/CSS

## 编码规范

- 遵循 PEP 8，统一使用 4 个空格缩进。
- Python 代码默认使用双引号，单行长度控制在 88 个字符以内。
- 业务逻辑优先放在 `models`、`forms`、`views` 中，不在模板里堆业务判断。
- Django CRUD 默认优先使用类视图，避免重复样板代码。
- 所有敏感信息通过环境变量注入，禁止写入仓库。
- 新增复杂逻辑时补最小必要注释，不写解释性废话注释。

## 输出约定

- 默认使用中文回复。
- 说明代码变更时，优先给出结果，再补充涉及的文件与验证方式。
- 生成代码时保持与当前项目目录结构一致，不额外引入未约定框架。
- 修改数据库相关内容时，同时检查 `docs/` 与 `schema/` 是否需要同步。
